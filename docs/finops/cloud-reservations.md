# Cloud Cost Savings with SavingPlans & Instance Reservation

## Problem:
- Cost-Saving Opportunity: Many cloud services used such as virtual machines, databases, and analytics platforms, were running on-demand, resulting in higher costs. However, workloads with predictable usage patterns presented an opportunity for significant savings through reserved instances. By committing to long-term usage of these services, it was possible to reduce cloud costs compared to on-demand pricing.

## Solution:
- Usage Analysis: Conducted a detailed analysis of historical cloud usage patterns across all services, identifying workloads suitable for reserved instances or capacity reservations.
- Reserved Azure Kubernetes Service (AKS) node pools for predictable workloads.
- Reserved Azure Database for PostgreSQL instances to stabilize database costs for long-term projects.
- Reserved Azure Files and Azure Storage reducing costs associated with high-volume storage and file sharing.
- Reserved Databricks reducing costs associated with processing data pipelines and analytics workloads.
- Dynamic Monitoring and Adjustments: Established a process for periodically reviewing reservation usage to adjust capacity or coverage based on evolving demands.
