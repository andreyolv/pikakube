# Scheduled Kubernetes Cluster Shutdown for Cost Optimization

## Problem:
- Underutilized Resources After Hours: Non-production Kubernetes clusters (e.g., dev, staging) remained fully active overnight and on weekends, even when no workloads or users were present.
- Cloud Cost Waste: Significant compute and storage resources were consumed during idle periods, leading to avoidable costs in development environments.
- No Native Scheduling: Cloud-native Kubernetes services like AKS, EKS, or GKE don’t provide built-in scheduling for start/stop operations, requiring custom automation.
