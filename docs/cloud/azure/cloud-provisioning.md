# Cloud Infrastructure Provisioning with IaC

## Problem:
- Resource Provisioning: Provisioning and managing diverse Azure services such as AKS, postgres, container registry, blob storage, file share, synapse, key vault, container app, event hubs, databricks, subnets.
- Standardization and Governance: Inconsistent provisioning across environments led to inefficiencies, security risks, and challenges with compliance.
- Network Security: Need to ensure resources are provisioned following security best practices, including access via private endpoints and private links.

## Solution:
- Infrastructure as Code (IaC): Used tools like Terraform and Crossplane to automate provisioning, ensuring consistent configurations, reducing errors, and speeding up deployments.
- Governance and Best Practices: Implemented naming conventions, policies, and tagging strategies to enhance clarity, maintainability, and security alignment.
- Network Security: Integrated private endpoints and private links to isolate services and ensure secure communication within Azure's private network.
- Resource Provisioning: Provisioning and managing diverse Azure services such Azure Kubernetes Service (AKS), Azure Database for PostgreSQL Flexible Server, Azure Container Registry (ACR), Azure Blob Storage, Azure File Share, Azure Synapse Analytics, Azure Key Vault, Azure Container Apps, Azure Event Hubs, Azure Databricks, Azure Subnets.
