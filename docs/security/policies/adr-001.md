# ADR-001: Kubernetes Secret Synchronization between Azure Key Vault and External Secrets Operator

**Status**: accepted  
**Deciders**: Andrey Oliveira 
**Consulted**: [Security Team, DevOps Team]  
**Date**: 2025-05-16  

---

## Context / Problem / Need

- Managing secrets in Kubernetes using tools like Sealed Secrets requires manual creation and rotation.
- Manual secret updates increase operational overhead and the risk of using outdated or compromised credentials.
- Organizations must comply with strict security and compliance standards, which demand centralized, secure, and auditable secret management.

---

## Requirements / Scope

- Automate synchronization of secrets from a secure external source into Kubernetes clusters.
- Eliminate manual effort and human error in secret management.
- Support automatic secret rotation.
- Ensure secrets are never exposed in version control.
- Provide auditability and access control.

---

## Considered Options

### 1. Sealed Secrets

**Pros**:

- Well-known in the Kubernetes community.
- GitOps-friendly; secrets are encrypted and stored in Git.

**Cons**:

- Secret rotation is manual.
- Operational burden increases with the number of secrets.
- High risk if secrets are not updated properly.

---

### 2. HashiCorp Vault Agent Injector

**Pros**:

- High level of security and flexibility.
- Native support for secret rotation.
- Fine-grained access control policies.

**Cons**:

- Requires operation and maintenance of Vault infrastructure.
- Steeper learning curve.
- Adds operational complexity and dependencies.

---

### 3. Azure Key Vault + External Secrets Operator (Chosen)

**Pros**:

- Native integration with Azure ecosystem.
- Supports automatic secret rotation.
- Easy to manage using Azure RBAC and auditing tools.
- External Secrets Operator dynamically keeps secrets up to date in Kubernetes.

**Cons**:

- Requires correct permissions setup in Azure.
- Depends on secure connectivity between Kubernetes and Azure Key Vault.

---

## Proof of Concept (PoC)

- Not necessary because Option 3 is good. 

---

## Decision / Solution

The chosen solution integrates **Azure Key Vault with Kubernetes via the External Secrets Operator**.  
This enables dynamic secret synchronization and automatic rotation, removing manual steps and improving security posture and compliance.

---

### Consequences / Technical Debt

**Positive**:

- Reduces operational overhead.
- Improves security and compliance.
- Centralized and auditable secret management.

**Negative / Debt**:

- Azure dependency — migration to another cloud provider may require reimplementation.
